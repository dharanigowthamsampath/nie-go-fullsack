### **MongoDB Indexes: A Complete Guide**  

Indexes in MongoDB **improve query performance** by allowing the database to quickly locate documents instead of scanning every document in a collection.  

---

## **1Ô∏è‚É£ What is an Index?**
An **index** is a data structure that stores a small portion of the collection‚Äôs data in a way that makes searching faster. It works like an index in a book, helping MongoDB find relevant documents quickly.

‚úÖ **Without an index:** MongoDB performs a **collection scan**, checking every document.  
‚úÖ **With an index:** MongoDB directly finds the required documents, improving speed.  

---

## **2Ô∏è‚É£ Creating an Index**
The `createIndex()` method is used to create indexes.  

### **Example: Creating an Index on One Field**
```js
db.users.createIndex({ name: 1 })
```
‚úÖ Creates an index on the `name` field in **ascending order (1)**.

---

## **3Ô∏è‚É£ Types of Indexes**  

### **1. Single Field Index**
Indexes one field in ascending (`1`) or descending (`-1`) order.

```js
db.users.createIndex({ age: -1 }) // Index on 'age' in descending order
```

---

### **2. Compound Index**
Indexes **multiple fields** together to optimize queries that filter by multiple fields.

```js
db.users.createIndex({ name: 1, age: -1 })
```
‚úÖ Optimized for queries filtering by `name` **and** `age`.

---

### **3. Unique Index**
Prevents duplicate values in a field.

```js
db.users.createIndex({ email: 1 }, { unique: true })
```
‚úÖ Ensures no two users have the same email.

---

### **4. Text Index (For Searching Strings)**
Used for **full-text search** in MongoDB.

```js
db.products.createIndex({ description: "text" })
```
‚úÖ Allows searching within the `description` field.

```js
db.products.find({ $text: { $search: "perfume" } }) // Find all products containing 'perfume'
```

---

### **5. Sparse Index**
Stores only documents where the indexed field **exists**, reducing storage.

```js
db.users.createIndex({ phone: 1 }, { sparse: true })
```
‚úÖ Only indexes documents that have a `phone` field.

---

### **6. TTL Index (Time-to-Live)**
Automatically deletes documents after a certain period.

```js
db.logs.createIndex({ createdAt: 1 }, { expireAfterSeconds: 3600 })
```
‚úÖ Deletes documents **1 hour** after `createdAt`.

---

## **4Ô∏è‚É£ Viewing and Dropping Indexes**
### **View All Indexes**
```js
db.users.getIndexes()
```

### **Drop a Specific Index**
```js
db.users.dropIndex("name_1")
```

### **Drop All Indexes**
```js
db.users.dropIndexes()
```

---

## **5Ô∏è‚É£ Example: How Indexes Improve Performance**
Consider a **users** collection with **1 million documents**:

üî¥ **Without an Index (Slow Query)**
```js

```
‚úÖ **MongoDB scans the entire collection** (very slow).

üü¢ **With an Index (Fast Query)**
```js

db.users.find({ age: 30 })
```
‚úÖ **MongoDB finds results instantly** using the index.

---

## **6Ô∏è‚É£ When to Use Indexes?**
‚úÖ Frequently queried fields (`name`, `email`, `category`).  
‚úÖ Sorting or filtering (`$sort`, `$match`).  
‚úÖ Searching large collections.  
‚úÖ Ensuring uniqueness (`unique: true`).  
‚úÖ Automatically deleting old data (`TTL Index`).
