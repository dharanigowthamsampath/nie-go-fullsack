# **Understanding Go Syntax and Rules**

## **Introduction**

Go (or Golang) is known for its simplicity, readability, and efficiency. It follows a minimalistic syntax with strict rules that make it easy to learn and use.

In this blog, we will explore the **basic syntax of Go** and discuss the **rules** you must follow when writing Go programs.

---

## **Basic Go Program Structure**

A typical Go program follows this structure:

```go filename="main.go" copy
package main      // 1. Define the package

import "fmt"      // 2. Import required packages

func main() {     // 3. Define the main function
    fmt.Println("Hello, Go!")  // 4. Program logic
}
```

Let‚Äôs break down the syntax and understand the rules for writing a Go program.

---

## **Go Syntax Rules**

### **1. Every Go Program Starts with a Package**

```go filename="main.go" copy
package main
```

- Go programs are organized into **packages**.
- The `main` package is mandatory for executable programs.
- If you‚Äôre creating a library or module, you can use other package names.

### **2. Importing Packages**

```go filename="main.go" copy
import "fmt"
```

- The `import` keyword is used to include built-in or third-party packages.
- **Example of importing multiple packages:**
  ```go filename="main.go" copy
  import (
      "fmt"
      "math"
  )
  ```

### **3. The Main Function is Required**

```go filename="main.go" copy
func main() {
}
```

- Execution starts from `main()`.
- Without `func main()`, your Go program won‚Äôt compile.

### **4. Statements End Without Semicolons (Mostly)**

- Unlike C or Java, Go **does not require semicolons (`;`)** at the end of statements.
- The compiler automatically inserts semicolons at the end of lines.
- However, in **one-line statements**, a semicolon is required:
  ```go filename="main.go" copy
  fmt.Println("Hello"); fmt.Println("Go")
  ```

### **5. Case Sensitivity**

- Go is **case-sensitive**, meaning `MyVariable` and `myvariable` are different.

### **6. Variable Declaration**

```go filename="main.go" copy
var name string = "Go"
```

- Use `var` to declare variables with an explicit type.
- Short-hand declaration (`:=`) can be used inside functions:
  ```go filename="main.go" copy
  name := "Go"
  ```

### **7. Functions Must Be Declared Properly**

```go filename="main.go" copy
func add(a int, b int) int {
    return a + b
}
```

- Functions must start with `func` followed by the function name.
- Parameters and return types must be specified.

### **8. Curly Braces `{}` Are Mandatory**

- In Go, `{}` **must** be on the same line as `if`, `for`, `func`, etc.
- ‚ùå Incorrect:
  ```go filename="main.go" copy
  func main()
  {
      fmt.Println("Wrong syntax")
  }
  ```
- ‚úÖ Correct:
  ```go filename="main.go" copy
  func main() {
      fmt.Println("Correct syntax")
  }
  ```

### **9. Indentation and Formatting Are Strict**

- Go has a built-in formatter:
  ```
  go fmt main.go
  ```
- Always format code using `gofmt` to ensure readability.

### **10. Single-line and Multi-line Comments**

- **Single-line comments** use `//`:
  ```go filename="main.go" copy
  // This is a single-line comment
  ```
- **Multi-line comments** use `/* */`:
  ```go filename="main.go" copy
  /*
     This is a multi-line comment.
     It spans multiple lines.
  */
  ```

## **Example: Writing a Program with Proper Go Syntax**

```go filename="main.go" copy
package main  // Defines the package

import "fmt"  // Imports fmt package for printing

// Main function - Entry point of the program
func main() {
    // Declaring and initializing a variable
    message := "Welcome to Go Syntax!"

    // Printing the message
    fmt.Println(message)
}
```

### **Output**

```
Welcome to Go Syntax!
```
