# **Pointers in Go**

## **Introduction**

A **pointer** in Go is a variable that stores the **memory address** of another variable.

- Instead of storing values directly, pointers **point to the location** of the value.
- Used to **modify values efficiently** and **avoid unnecessary copying** of large data.

---

## **Syntax of Pointers in Go**

```go filename="main.go" copy
var ptr *dataType
```

### **Explanation**

1. `*` (asterisk) before `dataType` â†’ Declares a **pointer variable**.
2. `ptr` â†’ The pointer variable.
3. `dataType` â†’ The type of data the pointer will store the address for.
4. **By default, pointers are `nil` if uninitialized.**

---

## **Rules for Pointers in Go**

âœ… **Use `&` (ampersand) to get the memory address of a variable.**  
âœ… **Use `*` (asterisk) to dereference (access the value at an address).**  
âœ… **Pointers store memory addresses, not actual values.**  
âœ… **Avoid modifying `nil` pointers** to prevent runtime errors.

---

## **Basic Example: Declaring and Using a Pointer**

### **Code**

```go filename="main.go" copy
package main
import "fmt"

func main() {
    x := 10
    ptr := &x // Pointer storing the address of x

    fmt.Println("Value of x:", x)        // Output: Value of x: 10
    fmt.Println("Address of x:", &x)     // Prints memory address
    fmt.Println("Pointer ptr:", ptr)     // Output: Address of x
    fmt.Println("Value at ptr:", *ptr)   // Output: 10 (dereferencing)
}
```

### **Explanation**

1. `x := 10` â†’ A normal integer variable.
2. `ptr := &x` â†’ Stores the **memory address** of `x` in `ptr`.
3. `fmt.Println(ptr)` â†’ Prints the memory address of `x`.
4. `fmt.Println(*ptr)` â†’ **Dereferencing**, retrieves the value stored at `ptr`.

---

## **Example: Modifying a Value Using a Pointer**

### **Code**

```go filename="main.go" copy
package main
import "fmt"

func main() {
    x := 20
    ptr := &x  // Pointer to x

    fmt.Println("Before modification:", x)  // Output: 20

    *ptr = 50 // Modifying value using pointer

    fmt.Println("After modification:", x)   // Output: 50
}
```

### **Explanation**

1. `ptr := &x` â†’ `ptr` stores the memory address of `x`.
2. `*ptr = 50` â†’ **Modifies `x` indirectly** through the pointer.
3. `x` changes from **20 to 50** because `*ptr` directly modifies it.

---

## **Example: Passing Pointers to Functions**

### **Code**

```go filename="main.go" copy
package main
import "fmt"

// Function to modify a value using a pointer
func updateValue(num *int) {
    *num = *num + 10
}

func main() {
    x := 40
    fmt.Println("Before function call:", x) // Output: 40

    updateValue(&x) // Passing pointer to function

    fmt.Println("After function call:", x)  // Output: 50
}
```

### **Explanation**

1. `func updateValue(num *int)` â†’ Function takes a **pointer to an int**.
2. `*num = *num + 10` â†’ **Dereferences** the pointer and modifies `x`.
3. `updateValue(&x)` â†’ Passes **memory address** to modify `x` directly.

## **Example: Using Pointers with Structs**

### **Code**

```go filename="main.go" copy
package main
import "fmt"

type Person struct {
    name string
    age  int
}

func updateAge(p *Person) {
    p.age = p.age + 1
}

func main() {
    p1 := Person{"Alice", 25}
    fmt.Println("Before:", p1.age) // Output: 25

    updateAge(&p1) // Passing struct pointer

    fmt.Println("After:", p1.age)  // Output: 26
}
```

### **Explanation**

1. `func updateAge(p *Person)` â†’ Accepts a pointer to a `Person` struct.
2. `p.age = p.age + 1` â†’ Directly modifies the original struct.
3. `updateAge(&p1)` â†’ **Passes address**, modifying `p1` in-place.
